# psyskill

创业者心理执行力工具箱。

不是治疗工具，不是激励工具，是**诊断 + 启动**工具。帮你看清楚为什么动不了，然后强制启动一个最小动作。

可在 Claude Code、Codex、Cursor、Trae Solo 等任意支持 skill / system prompt 的 Agent 上使用。

---

## 它解决什么问题

创业者每天要做的决定和动作，比上班族多 10 倍。但心理系统没有升级，就会反复出现这些状态：

- 知道该写方案、该见客户、该做交付，就是启动不了
- 方向换了一个又一个，每次都在「准备期」死掉
- 凌晨 1 点还在刷手机，白天靠咖啡硬撑
- 产品迭代永远在规划，永远不出第一个版本
- 一坐到电脑前就无意识刷社交媒体
- 有人看着效率高，一个人就废掉
- 身体很累但脑子停不下来，或者脑子很清醒但身体动不了

**这些问题，大部分不是意志力问题，是诊断错了层级。**

psyskill 的设计逻辑是：从身体到意义，从下到上排查，找到真正卡住你的那一层，然后直接启动动作。

---

## 设计哲学（继承自 dontbesilent）

1. **诊断优先于解决** —— 很多问题被看清楚之后就消失了，不需要解决
2. **先拆雷，再加速** —— 创业里最大的成本不是跑得慢，是方向错了还加速跑。psy 先帮你识别那些「看起来在努力，其实是在绕路」的行为
3. **元方法优先于具体方法** —— 不是给你 100 条建议，是给你一套思考框架
4. **诚实地承认边界** —— 明确告诉你什么是 skill 解决不了的

## 12 个 skill 一览

### 诊断与路由

| Skill | 做什么 | 什么时候用 |
|---|---|---|
| `/psy` | 主入口 + 智能路由。目标澄清 + ABC 行为分诊 + 层内信号识别 | 不知道用哪个工具，先说「我动不了」 |
| `/psy-goal` | 目标澄清。把模糊表达提纯成可验证目标 | 「我感觉我没把话说清楚」「我想先澄清目标」 |
| `/psy-diagnosis` | 综合诊断。一次过筛 7 层，出诊断报告 | 「帮我全面看看」「我不知道从哪开始」 |
| `/psy-first-principles` | 战略校准。用马斯克 5 步算法判断一件事值不值得做 | 「这件事值不值得做」「我是不是在优化错误的事」 |

### 行为设计

| Skill | 做什么 | 什么时候用 |
|---|---|---|
| `/psy-fogg` | 自动行为改造。用福格 B=MAP 改提示、降能力门槛、绑定习惯 | 「一不留神就刷手机」「习惯性地…」 |
| `/psy-science` | 长期习惯系统设计。用 SCIENCE 七个杠杆设计可坚持方案 | 「帮我设计坚持方案」「怎么让 XX 变成习惯」 |
| `/psy-context` | 情境校准。为什么 A 场景能动、B 场景不能 | 「在公司效率很高，回家就不行」 |

### 7 层心理排查

| Skill | 层级 | 解决什么 | 典型信号 |
|---|---|---|---|
| `/psy-body` | 生理层 | 睡眠、运动、激素、营养导致的「假执行力问题」 | 最近每天只睡 5 小时、完全没劲 |
| `/psy-meaning` | 存在层 | 这件事根本不该做，或动机错了 | 我不知道自己到底想不想做短视频 |
| `/psy-screen` | 深层心理层 | 识别创伤、依恋、需要专业治疗的情况 | 一打开文档就心慌，不知道怕什么 |
| `/psy-purpose` | 分诊层 | 阿德勒目的论：你不行动是为了什么 | 我知道该写脚本，但总是拖到最后一刻 |
| `/psy-thought` | 处理层 | CBT 认知扭曲识别与重构 | 我满脑子都是「这次肯定又没人看」 |
| `/psy-go` | 启动层 | 60 秒内让你动起来 | 别分析了，我现在就要开始 |

## 核心机制：诊断顺序不能反

绝大多数心理自助产品失败的原因，是允许用户自己选诊断方向。**用户会本能地避开真正的问题。**

psyskill 的排查顺序：

```
启动层 ← 校准层 ← 处理层 ← 分诊层 ← 深层心理层 ← 存在层 ← 生理层
                                                              ↑
                                                          从这里开始
```

**只有当下一层排除了，才上一层。任何跳级都是回避。**

但 `/psy` 主入口会先做一次快速分诊：如果某一层信号非常明确，直接定位到那一层，不用每层都走。

---

## 关键设计：强制启动

所有诊断 skill 完成后，**最后一句话必须是**：

> 现在立刻进入 `/psy-go`，不进则诊断作废。

**这是整套体系的扳机。**

没有这个机制，体系会变成「分析瘾」的助推器——用户分析得很爽，然后继续不动。

`/psy-go` 故意设计得不友好——冷、短、不共情。因为友好的启动器没用。市面上所有自我激励 app、习惯打卡、鸡汤推送都是友好的，效果都不行。

## 工具路径图

### 常见主线

```
方向对不对
    ↓
/psy-first-principles（用第一性原理拆一下）
    ↓
想不想做
    ↓
/psy-meaning（意义层排查）
    ↓
为什么做不动
    ↓
/psy-body → /psy-screen → /psy-purpose → /psy-thought → /psy-context
    ↓
启动
    ↓
/psy-go
```

### 横向工具

- `/psy-goal`：目标本身模糊，无法驱动行动
- `/psy-diagnosis`：多层信号混合，不知道问题在哪
- `/psy-fogg`：自动行为问题，靠意志力改不掉
- `/psy-science`：需要一套长期可坚持的执行系统
- `/psy-first-principles`：怀疑自己在优化错误的事

## 如何安装 psyskill

### 通用安装方式（适用于 Codex / Claude Code）

```bash
npx -y skills add SanLoveMatcha/sanpsy -g --all
```

> 这会从 GitHub 仓库拉取整套 skill 并安装到本地的 skill 目录。

### Claude Code

如果你用 Claude Code，推荐先用上面的 `npx skills add` 安装。安装后主入口就是 `/psy`。

### Trae Solo

Trae Solo 一个 zip 装一个 skill。进入仓库 `skills/` 目录，把每个子文件夹分别压缩成 zip，然后逐个拖进 Trae Solo 的「上传技能」窗口。

### 手动安装

把仓库 `skills/` 下的所有子文件夹复制到你的 skill 目录：

- Claude Code：`~/.claude/skills/`
- Codex：`~/.codex/skills/`
- 通用 Agents：`~/.agents/skills/`

每个子文件夹是一个独立 skill，主入口是 `/psy`。

## 几个关键词

- **ABC 行为分诊**：Sean Young《Stick with It》的框架，把「做不动」分成 Automatic（自动行为）、Burning（燃烧行为）、Common（普通行为）三类
- **7 层排查**：身体 → 意义 → 深层心理 → 目的 → 认知 → 情境 → 启动
- **强制启动**：诊断不结束于「看清楚」，结束于「立刻做一个物理动作」
- **先拆雷，再加速**：先识别「看起来在努力，其实是在绕路」的行为，再谈加速

---

## 这套体系不做什么

- **不替你完成动作。** 这是你的事。
- **不治疗抑郁、焦虑、双相、创伤后应激障碍。** 这些必须找专业的人。
- **不陪你走完改变的过程。** 一次诊断 + 一次启动是 skill 的边界，长期改变需要真人。
- **不让你「立刻有动力」。** 这种东西不存在。
- **不替代心理咨询师 / 治疗师 / 精神科医生。**

---

## 边界声明

这套 skill 是**自助工具**，不是医疗器械。

如果你出现以下情况，请立刻停止用 skill，去看专业的人：

- 持续 2 周以上情绪低落 + 兴趣丧失（疑似抑郁症）
- 持续焦虑伴随心慌、胸闷、惊恐发作
- 自伤念头或自杀念头
- 解离体验、闪回、严重躯体化反应
- 任何持续影响日常功能的状态

寻求专业帮助不是软弱，是判断力。
